### Image Upload Pattern

Images are uploaded to the `images` bucket and served via a public URL:

```javascript
async function uploadImage(file) {
  const ext = file.name.split('.').pop();
  const fileName = Date.now() + '_' + Math.random().toString(36).substring(2, 8) + '.' + ext;
  const result = await supabaseClient.storage.from('images').upload(fileName, file);
  if (result.error) return null;
  return 'https://exoyfpqfgixofsuginxx.supabase.co/storage/v1/object/public/images/' + fileName;
}

async function deleteImage(url) {
  const prefix = 'https://exoyfpqfgixofsuginxx.supabase.co/storage/v1/object/public/images/';
  if (!url || url.indexOf(prefix) !== 0) return;
  const path = url.substring(prefix.length);
  await supabaseClient.storage.from('images').remove([path]);
}
```

**Why single file?** Because CSS styles use globally-scoped class names that would collide with the main site's CSS, and JS functions share global state. Keeping everything inline eliminates dependency issues.

### Phase 2: Targeted Fixes

Once the build script approach was abandoned, direct fixes were applied to `admin.html`:

4. **`saveArticle()` null crash (line 391)** — The function used `document.getElementById('ate')` which returns `null` because the HTML form uses `id="article_title_en"`. Calling `.value.trim()` on `null` throws `TypeError`. Fix:
   ```javascript
   // Broken (crashes):
   var te = document.getElementById('ate').value.trim() || document.getElementById('article_title_en').value.trim();
   // Fixed (safe):
   var te = (document.getElementById('ate') || document.getElementById('article_title_en')).value.trim();
   ```
   The `||` fallback is checked **before** calling `.value.trim()`, so if `'ate'` doesn't exist, it uses the actual element. Applied to all 4 shorthand IDs: `ate`, `atg`, `ace`, `acg`.

5. **Article delete button** — Was calling `deleteArticle()` which doesn't exist. Changed to `da()`.

6. **Team photo delete button** — Had broken `''` patterns (empty string literal instead of escaped quotes):
   ```javascript
   // Broken:
   onclick="dtp('' + p.id + '','' + (p.image_url || '') + '')"
   // Fixed:
   onclick="dtp(\'' + p.id + '\',\'' + (p.image_url || '') + '\')"
   ```

7. **Messages mark-read button** — Same `''` pattern bug:
   ```javascript
   // Broken:
   mr('' + m.id + '')
   // Fixed:
   mr(\'' + m.id + '\')
   ```

8. **Messages delete button** — Same fix as mark-read.

9. **`renderSection()` function** — Was calling display names (`renderRoses`, `renderStaff`, etc.) that don't exist. The actual functions use minified names (`rr`, `rsf`, `rtp`, `rm`, `rcb`, `rset`). Fixed all 6 calls.

10. **`handleSectionAction()` function** — Was calling `showRoseForm()`, `showStaffForm()`, `showTeamPhotoForm()` which don't exist. Changed to `srf()`, `ssf()`, `stpf()`.

11. **Added 9 function aliases** at the bottom of the script tag to bridge minified names to their full implementations:

## 8. KEY PATTERNS & CONVENTIONS

### Language Switching (main site only)

The main site (`index.html`) supports Georgian (GE) and English (EN) via `js/language.js`. Elements with `data-lang-en` and `data-lang-ge` attributes get toggled. On page load, if `localStorage.getItem('lang') === 'ge'`, Georgian is shown; otherwise English.

### Image Handling

Images are uploaded to Supabase Storage bucket `images` with a unique filename (`timestamp_random.ext`). The public URL format is:
```
https://exoyfpqfgixofsuginxx.supabase.co/storage/v1/object/public/images/{filename}
```

### Toast Notifications

A global toast system exists in both admin.html and (if needed) can be replicated on the main site. Toasts auto-dismiss after 3 seconds:
```javascript
function showToast(message, type) {
  // types: 'success' (green), 'error' (red), 'info' (blue)
}
```

### Modal Pattern

Both admin and main site use an overlay + centered modal pattern:
```html
<div class="modal-overlay" id="modalOverlay">
  <div class="modal">
    <div class="modal-header"><h3 id="modalTitle">Title</h3><button onclick="closeModal()">&times;</button></div>
    <div class="modal-body" id="modalBody"></div>
  </div>
</div>
```

The overlay closes when clicking outside the modal (`e.target === this`).

### Confirmation Dialog

Used for destructive actions (delete article, delete staff, etc.):
```html
<div class="modal-overlay" id="confirmOverlay">
  <div class="modal" style="width:420px">
    <div class="confirm-dialog">
      <div class="icon">⚠️</div>
      <h3 id="confirmTitle">Are you sure?</h3>
      <p id="confirmMessage">This action cannot be undone.</p>

      </div>
    </div>
  </div>

```

Pattern: `showConfirm(title, message, asyncCallback)` sets up the dialog and stores the callback. On confirm button click, `executeConfirm()` runs the stored callback.

### Loading States

Buttons use a loading state via CSS classes:


## 9. TROUBLESHOOTING COMMON ISSUES

### "X is not defined"

This usually means a **function name mismatch** between what's called and what's defined. Check the function name mapping table in Section 4 above. If calling from an `onclick` attribute in a rendered HTML string, remember that the rendered string is HTML, not JS — you must use globally-scoped function names.

### `document.getElementById(...)` returns null

The form input field IDs in the modal HTML are shorthand (`ate`, `atg`, `ace`, `acg`) but the actual HTML uses long form (`article_title_en`, `article_title_ge`, etc.). Use the safe pattern:
```javascript
(document.getElementById('shortId') || document.getElementById('longId')).value.trim()
```

### Image upload returns null

Check that:
1. The Supabase `images` bucket is public
2. File size isn't too large (Supabase free tier limits vary)
3. The file type is an accepted image format

### Session doesn't persist

The admin uses `sessionStorage` which clears when the browser tab is closed. Use `localStorage` if persistence across tabs/sessions is needed (not recommended for security reasons — but this is a simple password check anyway).
